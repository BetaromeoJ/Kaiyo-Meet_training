Google Meetで実現する「学びの保障」

病気療養、不登校、家庭事情などにより教室で授業を受けることが難しい生徒にも、ICTを活用して学習機会を保障する。Meetで授業を届け、録音とAIで学びを残す一連の流れを、50分間で体験する研修です。

研修の基本方針

今回の研修では、Google Meetの多機能な使い方を広く紹介するのではなく、教室に来られない生徒にも授業を届けることを最優先にします。そのうえで、授業終了後に、Windowsのサウンドレコーダーで記録した授業音声をGeminiで整理し、授業のまとめとして残すところまでを一連の流れとして示します。

Purpose

研修の目的

病気療養、不登校、家庭事情などにより、教室で授業を受けることが難しい生徒に対しても、ICTを活用して学習機会を保障する方法を学ぶ。本研修では、次の2つを実現します。

① 今の学び

リアルタイムの学びを保障する

Google Meetを使い、教室で行われている授業を、自宅や別室にいる生徒へ届ける。

② 後からの学び

後からの学びを保障する

授業音声を記録し、Geminiで授業内容を整理して、出席した生徒にも欠席した生徒にも共有できる授業記録を作る。

info

文部科学省の資料でも、ICTを活用し、学校外での学習や、やむを得ず登校できない生徒への配慮を進める考え方が示されています。

研修終了時の到達目標

受講者が、次の内容を理解・体験できる状態を目指します。

  • check_circleGoogle Meetに参加できる
  • check_circleマイクとカメラを操作できる
  • check_circle教材を画面共有できる
  • check_circle教室と自宅をつなぐ授業の流れを理解できる
  • check_circleWindowsのサウンドレコーダーで授業音声を録音できる
  • check_circle録音データをGeminiに読み込ませ、授業内容を整理できる
  • check_circle生成した授業記録をGoogleドキュメントやClassroomで共有できる

録音時の注意

サウンドレコーダーは、基本的にSurfaceに接続されたマイクの音を録音します。そのため、研修では次の点を事前に確認します。

  • radio_button_unchecked教師の声が明瞭に録音されるか
  • radio_button_uncheckedMeet参加者の声がスピーカー経由で録音に入るか
  • radio_button_uncheckedピンマイクとMeetを同時に使用できるか
  • radio_button_uncheckedハウリングや音の反響が起きないか
  • radio_button_unchecked保存された音声をGeminiが読み込めるか
warning

Meet側の生徒の発言が明瞭に録音されない場合は、教師が質問を復唱します。

例:「オンライン参加の生徒から、資料3ページについて質問がありました。」

このように教師が言い直すことで、授業記録にも質問内容を残せます。

Gemini

録音した音声ファイルをGeminiにアップロードし、文字起こしと授業記録の作成を依頼します。

授業記録生成プロンプト

録音した授業音声をアップロードした後に入力します。

以下は高校の授業を録音した音声です。 音声を文字起こししたうえで、 内容を補いすぎず、次の形式で整理してください。 1. 授業のテーマ 2. 本時の目標 3. 授業で説明した内容 4. 重要なポイント 5. 生徒から出た質問と回答 6. 復習すべき内容 7. 次回までの課題 話し言葉の重複や言い直しを整理し、 高校生が読みやすい文章にしてください。

Program Flow

研修プログラムの流れ

全9ステップで、趣旨説明から模擬授業、録音・Gemini整理、共有、まとめまでを一続きに体験します。

提示するメッセージ:

今日の目的は、Google Meetの機能をすべて覚えることではありません。教室に来られない生徒にも、いつもの授業を届けられるようになることです。そのうえで、さらに授業音声を記録として残すことで、後から学び直す機会も保障します。

扱う機能は最小限にします。

ClassroomからMeetに参加 / マイクのオン・オフ / カメラのオン・オフ / チャット / 挙手機能 / 画面共有 / Meetからの退出

細かな設定や高度な機能は扱いません。

実物を見せながら説明します。黒板・スクリーンの前に教師(ピンマイク装着)、三脚で固定したタブレット、その先に生徒席という配置です。

ポイントは、機材を増やしすぎないことです。

Surfaceで実演します。

①サウンドレコーダーを開く　②入力マイクを確認する　③ピンマイクで短く話す　④録音を開始する　⑤録音中の画面を確認する　⑥その状態でMeetを始める

ここで、Meetより先にサウンドレコーダーを起動するという手順を標準化します。

参加者を2グループに分けます。教室グループは実際の教室で授業を受け、オンライングループは別室からMeetに参加します。

模擬授業では次の内容を体験します。デジタル教材の画面共有 / 教師の説明 / オンライン生徒への声かけ / チャットでの応答 / 挙手機能 / オンライン生徒からの質問 / 教師による質問の復唱

途中でグループを交代する場合は、短時間で行います。

①Google Meetを終了する　②サウンドレコーダーの録音を停止する　③音声ファイルを再生する　④教師とオンライン参加者の声が入っているか確認する　⑤保存場所を確認する

①Geminiを開く　②録音データをアップロードする　③用意したプロンプトを入力する　④文字起こしと授業記録を生成する　⑤内容を確認する

実演では、通信や処理時間に備え、事前に録音した短いサンプル音声も準備します。

Geminiが生成した内容を、教師が確認 → 必要な修正 → Googleドキュメントへ貼り付け → Classroomに投稿という流れで示します。

NotebookLMについては、次の一言に留めます。「この授業記録をNotebookLMのソースにすれば、復習資料や音声解説などに発展させることもできます。」

最後のスライドには、次の言葉を提示します。

「Meetで『今』の授業を届ける。録音とAIで『後から』の学びを残す。」

さらに、「まずは、教室に来られない生徒をMeetでつなぐところから始めてください。その授業を音声として残せば、参加できなかった生徒にも、学び直す資料を届けることができます。」と締めます。

Core Message

今回の研修で伝える中心メッセージ

この研修の主役は、サウンドレコーダーやGeminiではありません。主役はあくまでも、教室に来られない生徒の学びを、どう保障するかという課題です。

ツール役割
Google Meetリアルタイムで授業を届ける
デジタル教材教室と自宅に同じ教材を提示する
サウンドレコーダー授業音声を記録する
Gemini授業音声を文字起こし・整理する
Googleドキュメント授業記録として保存する
Google Classroom生徒へ配布する

研修全体を、次の3つの言葉で整理すると、先生方にも伝わりやすくなります。

つなぐ 残す 届け直す

Meetで「今」の授業を届ける。
録音とAIで「後から」の学びを残す。

まずは、教室に来られない生徒をMeetでつなぐところから始めてください。その授業を音声として残せば、参加できなかった生徒にも、学び直す資料を届けることができます。

Privacy

個人情報の取扱い

warning

録音した授業音声や生成した授業記録には、生徒の氏名や発言内容が含まれます。Geminiへアップロードする際は、県域教育アカウントの利用範囲・保存先を確認し、不要になった録音データは速やかに削除してください。生成された授業記録は、必ず教師が内容を確認したうえで共有してください。
